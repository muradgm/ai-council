
# Release Readiness Engine Scoring Rubric

Score each criterion from **1 to 5**.

- **1** = unacceptable, unclear, or high-risk.
- **2** = weak; major gaps.
- **3** = usable; some risk remains.
- **4** = strong; minor gaps.
- **5** = excellent; evidence-backed and robust.

| Criterion | 1 | 3 | 5 | Notes |
|---|---|---|---|---|
| Critical Bugs | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Rollback Plan | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Docs Updated | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Monitoring Ready | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Support Plan | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Security Checks | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Business Acceptance | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |

## Scoring rules

- Do not hide critical flaws behind a high average.
- Mark confidence separately from score.
- Penalize unsupported claims.
- Penalize solutions that require unrealistic discipline, budget, or data.
- Prefer reversible options when evidence is thin.
