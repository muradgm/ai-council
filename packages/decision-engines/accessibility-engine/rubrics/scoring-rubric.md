
# Accessibility Engine Scoring Rubric

Score each criterion from **1 to 5**.

- **1** = unacceptable, unclear, or high-risk.
- **2** = weak; major gaps.
- **3** = usable; some risk remains.
- **4** = strong; minor gaps.
- **5** = excellent; evidence-backed and robust.

| Criterion | 1 | 3 | 5 | Notes |
|---|---|---|---|---|
| Semantic Structure | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Keyboard Usability | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Contrast | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Focus States | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Error Messages | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Assistive Tech Support | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Inclusive Interaction | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |

## Scoring rules

- Do not hide critical flaws behind a high average.
- Mark confidence separately from score.
- Penalize unsupported claims.
- Penalize solutions that require unrealistic discipline, budget, or data.
- Prefer reversible options when evidence is thin.
