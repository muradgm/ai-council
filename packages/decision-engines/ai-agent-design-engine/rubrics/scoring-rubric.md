
# AI Agent Design Engine Scoring Rubric

Score each criterion from **1 to 5**.

- **1** = unacceptable, unclear, or high-risk.
- **2** = weak; major gaps.
- **3** = usable; some risk remains.
- **4** = strong; minor gaps.
- **5** = excellent; evidence-backed and robust.

| Criterion | 1 | 3 | 5 | Notes |
|---|---|---|---|---|
| Role Clarity | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Tool Boundaries | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Context Loading | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Failure Handling | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Evaluation Plan | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Safety Limits | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Cost Control | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |

## Scoring rules

- Do not hide critical flaws behind a high average.
- Mark confidence separately from score.
- Penalize unsupported claims.
- Penalize solutions that require unrealistic discipline, budget, or data.
- Prefer reversible options when evidence is thin.
