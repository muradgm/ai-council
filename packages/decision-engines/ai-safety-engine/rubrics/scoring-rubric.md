
# AI Safety Engine Scoring Rubric

Score each criterion from **1 to 5**.

- **1** = unacceptable, unclear, or high-risk.
- **2** = weak; major gaps.
- **3** = usable; some risk remains.
- **4** = strong; minor gaps.
- **5** = excellent; evidence-backed and robust.

| Criterion | 1 | 3 | 5 | Notes |
|---|---|---|---|---|
| Harm Potential | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| User Stakes | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Autonomy Level | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Data Sensitivity | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Misuse Risk | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Human Review Need | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Guardrail Clarity | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |

## Scoring rules

- Do not hide critical flaws behind a high average.
- Mark confidence separately from score.
- Penalize unsupported claims.
- Penalize solutions that require unrealistic discipline, budget, or data.
- Prefer reversible options when evidence is thin.
