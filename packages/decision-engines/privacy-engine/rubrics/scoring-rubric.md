
# Privacy Engine Scoring Rubric

Score each criterion from **1 to 5**.

- **1** = unacceptable, unclear, or high-risk.
- **2** = weak; major gaps.
- **3** = usable; some risk remains.
- **4** = strong; minor gaps.
- **5** = excellent; evidence-backed and robust.

| Criterion | 1 | 3 | 5 | Notes |
|---|---|---|---|---|
| Data Minimization | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Purpose Limitation | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Retention Clarity | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Access Controls | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Deletion Path | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| User Transparency | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |
| Auditability | 1 = weak / unclear | 3 = acceptable | 5 = strong / evidence-backed | |

## Scoring rules

- Do not hide critical flaws behind a high average.
- Mark confidence separately from score.
- Penalize unsupported claims.
- Penalize solutions that require unrealistic discipline, budget, or data.
- Prefer reversible options when evidence is thin.
