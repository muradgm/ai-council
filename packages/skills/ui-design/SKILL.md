# UI Design Skill

## Purpose

This skill helps AI Council agents perform interfaces, layouts, visual systems with repeatable standards, explicit reasoning, and documented output.

## When to use

Use this skill when the request involves:

- ui design planning
- ui design review
- ui design execution or documentation

## Operating principles

1. Start by clarifying the real objective, not only the visible task.
2. Identify assumptions, constraints, risks, and missing context.
3. Prefer simple, durable solutions over clever but fragile ones.
4. Produce reusable artifacts: decisions, templates, checklists, code, or documentation.
5. Record tradeoffs when there is more than one reasonable path.

## Standard workflow

1. **Understand** — summarize the goal, stakeholders, constraints, and success criteria.
2. **Diagnose** — inspect the current state, identify gaps, and check for hidden risks.
3. **Plan** — propose a small number of viable options and choose the strongest one.
4. **Execute** — produce the artifact, implementation, analysis, or decision.
5. **Review** — test against quality standards and known failure modes.
6. **Document** — update relevant docs, examples, or decision records.

## Quality bar

A good output from this skill is:

- specific enough to act on,
- honest about uncertainty,
- grounded in project context,
- structured for future reuse,
- safe under realistic failure conditions.

## Common mistakes

- Treating symptoms as the problem.
- Creating isolated deliverables that do not fit the system.
- Skipping risk review.
- Over-optimizing before validating demand or correctness.
- Producing generic advice instead of a concrete next action.

## Deliverables

Typical deliverables include:

- decision memo,
- implementation plan,
- audit report,
- checklist,
- template,
- review notes,
- test or validation plan.

## Safety and boundaries

This skill should remain within the available evidence and project constraints. Escalate uncertainty instead of inventing facts.
