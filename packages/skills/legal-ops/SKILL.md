# Legal Ops Skill

## Purpose
This skill gives AI Council agents a reusable operating method for **Legal Ops** work. It is designed for practical execution, not generic advice.

## When to Use
Use this skill when the task involves:
- Contracts
- Compliance awareness
- Privacy
- IP
- Escalation

## Inputs Required
- User objective
- Constraints and deadline
- Current state or existing artifact
- Success criteria
- Known risks or non-goals

## Operating Principles
- Start from the user's concrete goal, not from a generic framework.
- Challenge weak assumptions before producing recommendations.
- Prefer small, testable next steps over vague strategic language.
- Separate facts, assumptions, hypotheses, and opinions.
- Produce artifacts that can be reviewed, reused, or implemented.

## Workflow
1. **Clarify the job**: define the decision, deliverable, or outcome.
2. **Map assumptions**: list what must be true for the plan to work.
3. **Inspect constraints**: time, budget, skill, legal, safety, tools, data.
4. **Generate options**: minimum three credible paths when strategy is involved.
5. **Evaluate trade-offs**: cost, upside, risk, reversibility, effort.
6. **Select path**: recommend one route and explain why.
7. **Produce deliverable**: template, plan, review, checklist, code, or brief.
8. **Run quality gate**: check clarity, correctness, completeness, risk.

## Quality Standards
- Specific over broad.
- Evidence-aware over confident guessing.
- Actionable over inspirational.
- Reusable structure over one-off response.
- Clear risk boundaries, especially in legal, financial, health, or security contexts.

## Common Failure Modes
- Treating the user's first framing as correct.
- Producing a polished answer without testing assumptions.
- Giving strategic advice without an execution path.
- Ignoring downside risk.
- Mixing personal opinion with evidence.

## Output Formats
- Strategy memo
- Audit report
- Execution plan
- Checklist
- Decision table
- Review notes
- Implementation brief

## Checklist
- [ ] Objective is explicit.
- [ ] Assumptions are listed.
- [ ] Risks are stated.
- [ ] Recommendation is concrete.
- [ ] Next step is actionable.
- [ ] Output can be reused by another agent.
