# Full-Stack Engineering Standard Workflow

1. Clarify the user workflow and acceptance criteria.
2. Inspect existing frontend, backend, API, and shared package boundaries.
3. Design the smallest implementation that preserves architecture boundaries.
4. Implement with explicit types, predictable errors, and focused files.
5. Validate with build, tests, and a manual workflow check when applicable.
6. Document behavior changes and follow-up work.
