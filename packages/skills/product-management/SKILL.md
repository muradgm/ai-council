# Product Management

## Purpose

Translate user problems into product strategy, feature scope, prioritization, and delivery plans.

## When To Use

Use this skill when the task requires roadmapping, feature planning, prioritization, PRDs, or product reviews.

## Inputs

- User goal
- Project context
- Constraints
- Existing repo files
- Required output format

## Outputs

- Practical recommendation or implementation plan
- Clear assumptions
- Quality checklist
- Risks and limitations

## Core Principles

- Prefer evidence over opinion.
- Make assumptions explicit.
- Optimize for useful output, not impressive language.
- Respect project constraints and existing architecture.
- Document decisions that future agents need to understand.

## Workflow

1. Define the problem precisely.
2. Identify constraints and success criteria.
3. Review existing project context.
4. Select an approach.
5. Produce the deliverable.
6. Review against the checklist.
7. State risks, tradeoffs, and next steps.

## Quality Bar

The result should be specific, grounded, reusable, and easy for another agent or developer to continue.

## Common Failure Modes

- Generic advice without project context.
- Duplicating existing repo concepts.
- Overengineering.
- Ignoring risk boundaries.
- Producing output without a clear next action.

## Safety / Risk Notes

Do not overbuild. Define the smallest useful version first.

## Checklist

- [ ] Problem is clearly defined.
- [ ] Constraints are listed.
- [ ] Existing repo context was considered.
- [ ] Output is actionable.
- [ ] Risks are stated.
