# Domain Safety Rubric

High-risk domains require stricter review.

| Domain | Required controls |
|---|---|
| Trading/finance | risk sizing, no guarantees, backtesting caveats |
| Security | no harmful exploitation guidance, defensive framing |
| Legal | jurisdiction caveat, encourage professional advice |
| Health-adjacent | medical caveat, encourage clinician review |
| Data/privacy | privacy impact, retention, consent, security controls |
