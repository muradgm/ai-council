# Routing Rubric

A route is strong when it selects:

1. The right senior agent owner.
2. At least one relevant decision engine for non-trivial decisions.
3. The smallest useful set of skills.
4. A workflow matching the work type.
5. A deliverable template when output needs structure.

## Failure patterns

- Selecting a generic agent when a specialized one exists.
- Loading too many unrelated skills.
- Missing risk/security review for high-risk work.
- Selecting implementation workflow before discovery or architecture is needed.
