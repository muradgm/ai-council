# Risky Actions Policy

## High-risk examples

- Delete, overwrite, or move user data.
- Run shell commands that mutate the filesystem outside a generated workspace.
- Send email, publish posts, commit to remotes, deploy, or create paid resources.
- Add or reveal credentials.
- Connect private project data to cloud AI providers without permission.
- Generate real trading instructions or live-trading automation.

## Safe alternative

When risk is high, produce a plan, patch, draft, or review checklist first.
