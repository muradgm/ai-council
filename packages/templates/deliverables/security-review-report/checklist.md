# Security Review Report Quality Checklist

Before delivery, verify:

- [ ] The template selected is appropriate for the user's request.
- [ ] Required inputs are present or assumptions are stated.
- [ ] The selected agents are relevant: cybersecurity-engineer, legal-risk-spotter, senior-backend-engineer.
- [ ] The selected skills are relevant: security, legal-basics, backend.
- [ ] The decision engines are relevant: security-review-engine, privacy-engine, legal-compliance-engine.
- [ ] The output includes risks, constraints, and open questions.
- [ ] The output is specific enough to execute or review.
- [ ] No unsupported claim is presented as fact.
