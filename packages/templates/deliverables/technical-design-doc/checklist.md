# Technical Design Doc Quality Checklist

Before delivery, verify:

- [ ] The template selected is appropriate for the user's request.
- [ ] Required inputs are present or assumptions are stated.
- [ ] The selected agents are relevant: principal-software-architect, senior-backend-engineer, cto.
- [ ] The selected skills are relevant: software-architecture, backend, database.
- [ ] The decision engines are relevant: architecture-engine, api-design-engine, database-engine.
- [ ] The output includes risks, constraints, and open questions.
- [ ] The output is specific enough to execute or review.
- [ ] No unsupported claim is presented as fact.
