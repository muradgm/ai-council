# Roadmap: SignalScout

## Phase A — Clarify MVP

- Confirm the smallest valuable workflow.
- Define success criteria.
- Create PRD and technical spec.
- Confirm data model.

## Phase B — Build Thin Slice

- Implement the core user input.
- Implement the main recommendation/review/result output.
- Add explainability.
- Add basic persistence.

## Phase C — Review + Iterate

- Run product review.
- Run architecture review.
- Run UX/UI review.
- Test with realistic sample data.

## Phase D — Release Candidate

- Add onboarding.
- Add export/reporting if needed.
- Run quality gates.
- Update docs and memory.

## Later

- Add integrations only after the core workflow proves useful.
