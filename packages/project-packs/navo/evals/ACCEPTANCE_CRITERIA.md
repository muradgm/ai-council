# Acceptance Criteria: NAVO / Flowday

## Product Acceptance

- The core workflow is understandable without explanation.
- The output clearly supports the wedge.
- The user can recover from mistakes.
- The system explains recommendations or reviews.

## Technical Acceptance

- Existing conventions are reused.
- Core behavior is testable.
- Data model is documented.
- Risky actions require explicit review.
- Docs and project memory are updated.
