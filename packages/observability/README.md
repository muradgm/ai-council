# Observability Package

The observability package gives AI Council a runtime record of what happened, why it happened, which tools and providers were involved, what artifacts were produced, and what risks or costs were introduced.

This is not a cloud monitoring replacement. It is a local-first, repository-native trace and evidence layer designed for Codex, CLI agents, and human review.

## Core responsibilities

- Record council runs and workflow executions.
- Record provider calls without storing private prompts by default.
- Track artifacts generated by the Council.
- Track estimated provider cost and token usage.
- Produce diagnostics reports.
- Support incident review and regression analysis.
- Give future agents enough evidence to understand previous decisions.

## Storage

Runtime records are written under:

```text
storage/observability/
├── runs/
├── traces/
├── provider-calls/
├── costs/
├── artifacts/
├── diagnostics/
├── reports/
└── incidents/
```

## Rule

Observability must improve accountability without turning into surveillance. Store summaries, metadata, hashes, and redacted excerpts unless the user explicitly asks for full trace content.
