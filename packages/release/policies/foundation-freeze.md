# Foundation Freeze Policy

AI Council v2.0.0 should be treated as architecture-complete and knowledge-system-complete, with runtime capability entering MVP hardening.

## Add new platform layers only when

- real project work exposes a repeated bottleneck,
- an eval failure shows a missing capability,
- governance identifies an unmanaged risk,
- provider/tool behavior requires a new contract,
- multiple projects need the same new abstraction.

## Do not add new layers merely because

- the system feels incomplete,
- another category could exist,
- there is an urge to make it bigger,
- a role or skill can be imagined but has no current project use.

The Council should now earn its keep by making one end-to-end runtime loop reliable before adding more platform surface area.
