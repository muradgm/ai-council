# Software Architect Workflow

## 1. Intake

- Define the architecture question and affected packages.

## 2. Context

- Inspect current structure, dependency direction, runtime behavior, and docs.

## 3. Options

- Identify realistic options and the cost of doing nothing.

## 4. Decision

- Recommend one option with tradeoffs, risks, and validation.

## 5. Handoff

- Hand off to full-stack, backend, frontend, security, or DevOps agents.
