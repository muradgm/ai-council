# 06 Review

## Goal

Run quality gates, verify outputs, record risks, and update documentation.

## Actions

- Read `workflow.json` and `WORKFLOW.md`.
- Check the current project context before creating new files.
- Use only the agents, skills, and engines that materially improve the outcome.
- Capture assumptions and evidence.
- Keep output specific, actionable, and testable.

## Exit criteria

- The phase produces a concrete artifact or decision.
- The next phase can proceed without guessing.
- Any unresolved ambiguity is documented as a risk or open question.
