# Local / Cloud Policy

## Use local

- private source code;
- unpublished product strategy;
- credentials or logs;
- personal data;
- early rough drafts.

## Use cloud

- public information;
- sanitized code snippets;
- architecture reviews where user approves external reasoning;
- high-quality final critique;
- multimodel consensus.

## Mixed mode

For most projects, use local models for first-pass extraction and cloud models for sanitized strategic review.


# Provider Fallback Policy

Fallback order:

1. Same tier alternate provider.
2. Lower-cost provider if quality risk is low.
3. Premium escalation if quality risk is high.
4. Human handoff if the result remains uncertain.

Do not silently downgrade quality for high-risk tasks. Downgrades must be explicit in the final report.
