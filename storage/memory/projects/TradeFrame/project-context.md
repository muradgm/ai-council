# Project Context: TradeFrame

## Purpose

AI-first trading journal and decision review system.

## Product Wedge

Risk-first trading journal focused on decision quality, rule discipline, and post-trade learning rather than prediction hype.

## Current Stage

planning / MVP definition.

## Target Users

- retail forex traders
- multi-asset discretionary traders
- traders trying to reduce repeated mistakes

## Core Problem

Traders often record trades too late, focus only on profit/loss, and miss the behavioral, risk, and process errors that actually drive bad outcomes.

## MVP Scope

- trade capture
- pre-trade checklist
- post-trade review
- risk rule tracking
- mistake tagging
- weekly review report
- decision quality score

## Non-Negotiable Constraints

- Never present analysis as financial advice.
- Prioritize risk review, rule adherence, and journaling over prediction.
- Use explicit disclaimers for trading outputs.
- Avoid live execution features in early phases.
